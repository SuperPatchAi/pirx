# The Science Behind PIRX

## Intent

This source note captures what the science paper must communicate and how it is rendered.
The final PDF is generated by `docs/build_science_pdf.py` using ReportLab and follows the
academic narrative style used in the reference studies.

## Required Narrative Structure

- Abstract
- Introduction
- Methods
- Results
- Discussion
- Limitations
- Conclusion
- References

Target length: **12-15 pages** (two-column academic format).

## Required Content Coverage

1. **What PIRX calculates**
   - Activity cleaning gates and validity filters.
   - Feature engineering windows and weighted recency model.
   - Projection equations (driver weighting, improvement cap, volatility dampening).
   - Supported-range uncertainty calculations.
   - Event scaling equations and exponent bounds.
   - Readiness and injury-risk additive logic.

2. **Why PIRX is built this way**
   - Deterministic-first projection path as reliability floor.
   - Rollout-gated ML serving to prevent uncontrolled model drift.
   - Explainability and provenance metadata (`model_source`, `model_confidence`, `fallback_reason`).
   - Safety rationale for structural shift thresholding and fallback behavior.

3. **ML implementation status**
   - Implemented and active
   - Implemented but rollout-gated
   - Implemented but non-primary

4. **Figures and tables**
   - Pipeline diagram
   - Serving orchestration/fallback diagram
   - Uncertainty composition diagram
   - Constants/threshold table
   - ML component status table
   - Formula catalog appendix
   - Rollout/monitoring signals appendix

## Build

Run:

```bash
./docs/build_science_pdf.sh
```
