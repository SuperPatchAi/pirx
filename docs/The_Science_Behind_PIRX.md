# The Science Behind PIRX

## Intent

This source note captures what the science paper communicates and how it is rendered.
The final PDF is generated by `docs/build_science_pdf.py` using ReportLab and follows
academic paper conventions (author block, abstract, keywords, numbered equations,
figure/table captions, in-text citations, and references).

## Document Structure

1. Title page (single-column): title, subtitle, authors, abstract, keywords
2. Introduction with related work
3. System Architecture (pipeline, technology stack)
4. Data Conditioning (cleaning rules, pace normalization, design rationale)
5. Feature Engineering (temporal weighting, 5 feature domains, ACWR with EWMA seed, pattern detection, design rationale)
6. Baseline Estimation (tiered fallback with Riegel scaling, race detection, KNN cold-start, design rationale)
7. Projection Engine (driver scoring with ACWR U-shape, weighted aggregation, improvement, dampening, range, decomposition with even-distribution edge case, structural shift, GB per-user model, LSTM temporal model, constants, design rationale)
8. Event Scaling (Riegel, volume-adjusted exponent, individual fit, 5K transition, environmental, design rationale)
9. Readiness and Injury Risk (5-component readiness with GBC classifier, ACWR balance, freshness, recency, physiology, consistency, two-tier injury risk RF with calibration, DTW-based trajectory with heuristic fallback, design rationale)
10. Model Orchestration and Rollout (3-path serving: deterministic/GB/LSTM, Optuna HPO, lifecycle, endpoints, observability, promotion)
11. Operational Safeguards (inactivity decay, confidence, accuracy, bias correction, task scheduling, validation methodology)
12. API-Level Derived Calculations (driver persistence, fallbacks, zone methodology with Polarized/Pyramidal/Threshold-heavy/Mixed, economy, pattern detection with KMeans + correlation)
13. Discussion
14. Limitations
15. Future Work
16. Conclusion
17. Appendices (A: Formula Catalog, A2: Module Map, B: Status Matrix, C: Glossary, D: Non-Primary Modules + SHAP TreeExplainer, E: Edge-Case Policies, F: Rollout Scenarios)
18. References

## Metrics

- **Target length:** 14-18 pages (two-column academic format)
- **Current:** ~16 pages
- **Equations:** 87 numbered

## Content Coverage

1. **What PIRX calculates** — every formula from `README.md` sections 1-9, with equation numbers, constants tables, and cross-references.
2. **Why PIRX is built this way** — design rationale sections for features, baseline, projection, scaling, readiness, and rollout.
3. **ML implementation status** — explicit labels (active, rollout-gated, non-primary) for all modules.
4. **Trained ML models** — per-user Gradient Boosting (sklearn), LSTM (PyTorch), GBC readiness, RF injury risk, SHAP TreeExplainer, K-Means clustering, DTW trajectory matching, Optuna HPO.
5. **Figures and tables** — pipeline diagram, serving topology (3-path), uncertainty composition, 26 data tables.
